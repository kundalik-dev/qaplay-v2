FAQ

Why does clicking a disabled button not throw an error in Playwright? ⌄
Playwright's click() by default checks actionability — it waits for the element to be enabled. On a disabled button it will timeout. Use toBeDisabled() assertion instead to verify the state.
How do I perform a click-and-hold in Selenium? ⌄
Use the Actions class: new Actions(driver).clickAndHold(element).pause(1500).release().perform()
What's the difference between click() and submit() in Selenium? ⌄
click() simulates a user click on any element. submit() only works on form elements and submits the enclosing form. Prefer click() for button automation.

Button Automation Practice

Master button interactions in Selenium & Playwright — click, double-click, right-click, disabled state, click-and-hold.

🌱 Beginner ⏱ 10 min ⚡ 8 scenarios

Buttons are the most commonly interacted element in web automation. Engineers must handle: single click — the basic click() action, double click — fires a different browser event, right click — opens the context menu, and disabled button — assert it cannot be interacted with.

Selenium uses the Actions class for advanced mouse interactions. Playwright has click options built in.

1 Single Click

The most fundamental action. Use element.click() in Selenium or locator.click() in Playwright. Both trigger a full mouse press and release cycle.

7 FAQ

Why does Playwright timeout on a disabled button click? ⌄
Playwright performs actionability checks before clicking — a disabled element fails these checks and the call times out after the configured timeout period. Assert toBeDisabled() instead.
What's the difference between click() and dblclick()? ⌄
click() fires mousedown, mouseup, and click events. dblclick() fires two rapid click cycles followed by a dblclick event. The browser treats them as distinct interactions.
How do I target buttons with stable selectors? ⌄
Prefer data-testid attributes. QA Playground adds these to every practice element. In Playwright use page.getByTestId('btn-id'). In Selenium use By.cssSelector("[data-testid='btn-id']").
Does Cypress support right-click? ⌄
Yes. Use cy.get(sel).rightclick(). This fires the contextmenu event on the element, which is what automation needs to assert.
