→Overview

Buttons are the most commonly interacted element in web automation. Engineers must handle four core interactions: single click, double click, right click, and asserting disabled state.

Selenium uses the Actions class for advanced mouse interactions. Playwright has all click options built into locator.click(). Cypress has dedicated methods for each click type.

1Single Click

The most fundamental button action. Fires mousedown, mouseup, and click events in sequence. All three frameworks map directly to the same underlying browser event.

2Double Click

A double-click fires a dblclick DOM event. It is distinct from two rapid single clicks — the browser coalesces them into a single dblclick event. Selenium requires the Actions class; Playwright exposes locator.dblclick() directly.

3Right Click

A right-click fires a contextmenu event. Browsers usually show a native context menu; automation intercepts the event before that happens. Playwright uses { button: 'right' }; Selenium uses contextClick(); Cypress has rightclick().

4Disabled State

A disabled button has the HTML disabled attribute. The automation goal is to assert this state — not to click it. Playwright will timeout if you attempt to click a disabled element because it fails actionability checks.

5Click and Hold

Some UI patterns require holding a mouse button for a duration before releasing — sliders, drag handles, long-press menus. Selenium uses clickAndHold() with pause(). Playwright uses mouse.down() with waitForTimeout().

7FAQ

Why does Playwright timeout when clicking a disabled button? ⌄
Playwright performs actionability checks before clicking — it waits for the element to be visible, stable, enabled, and in the viewport. A disabled button fails the "enabled" check and the call times out after the configured timeout. Use toBeDisabled() to assert the state instead.
What's the difference between click() and dblclick()? ⌄
click() fires mousedown → mouseup → click events. dblclick() fires two rapid click cycles followed by a dblclick event. The browser treats them as distinct — a double-click does not fire two independent click events at the application level.
What is the most stable selector strategy for buttons? ⌄
Prefer data-testid attributes — they are stable, independent of styling, and survive refactoring. QA Playground adds data-testid to every practice element. In Playwright use page.getByTestId('btn-id'). In Selenium use By.cssSelector("[data-testid='btn-id']").
How do I perform click-and-hold in Selenium Java? ⌄
Use the Actions chain: new Actions(driver).clickAndHold(element).pause(Duration.ofMillis(1500)).release().perform(). Always call perform() at the end — the chain builds the action but does not execute until then.
Does Cypress support right-click natively? ⌄
Yes, since Cypress 6.1. Use cy.get(selector).rightclick(). This fires the contextmenu event on the element. Note that the native browser context menu will not appear — Cypress suppresses it. Your app's custom context menu handler will still run.
