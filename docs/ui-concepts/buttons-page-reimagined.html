Overview

Buttons are the most fundamental interactive element you'll automate. This page covers 6 core interaction types — from simple single clicks to advanced hold actions — across Playwright, Selenium, and Cypress.

1 · Single Click

The most common automation action. All three frameworks click and auto-wait for the element to be actionable.

TypeScript 
// Playwright — auto-waits for actionability
await page.locator('#clickBtn').click();
await expect(
  page.locator('#clickResult')
).toBeVisible();
2 · Double Click

A single click won't trigger double-click events. Use the dedicated API in each framework.

TypeScript 
// Playwright
await page.locator('#doubleClickBtn').dblclick();
await expect(
  page.locator('#doubleClickResult')
).toHaveText('Double clicked!');
3 · Right Click

Triggers context menus. Selenium uses Actions API; Playwright and Cypress have first-class support.

TypeScript 
// Playwright
await page.locator('#rightClickBtn')
  .click({ button: 'right' });
await expect(
  page.locator('#contextMenu')
).toBeVisible();
4 · Disabled State

Assert the button is disabled before trying to interact. In Playwright, clicking a disabled button times out after the actionability check.

TypeScript 
// Playwright — assert disabled, don't click
await expect(
  page.locator('#disabledBtn')
).toBeDisabled();
5 · Text Change

Verifying that a button's label or an associated element's text changes after a click.

TypeScript 
// Playwright — capture before, compare after
const btn = page.locator('#toggleBtn');
const before = await btn.textContent();
await btn.click();
expect(await btn.textContent()).not.toBe(before);
6 · Keyboard Enter

Keyboard activation is required for accessible automation. A focused button responds to Enter and Space the same as a click.

TypeScript 
// Playwright
await page.locator('#submitBtn').press('Enter');

FAQ
How do I click a button using Selenium WebDriver? ⌄
Locate the element with a stable selector and call click() — e.g. driver.findElement(By.cssSelector("[data-testid='btn-navigate-home']")).click();. Prefer data-testid or id over brittle XPath. Selenium scrolls the element into view automatically before clicking.
How do I perform a double-click in Selenium WebDriver? ⌄
A plain click() cannot express a double click — use the Actions API: new Actions(driver).doubleClick(element).perform();. In Playwright use locator.dblclick() and in Cypress cy.get(sel).dblclick().
How do I right-click a button to trigger a context menu? ⌄
Use new Actions(driver).contextClick(element).perform();. Playwright: locator.click({ button: 'right' }); Cypress: cy.get(sel).rightclick(). If the app renders a custom menu, assert it becomes visible afterwards.
How do I check if a button is disabled before clicking it? ⌄
Assert the state instead of clicking. Selenium: element.isEnabled() returns false for disabled buttons. Playwright: await expect(locator).toBeDisabled(). Clicking a disabled button in Playwright throws after the actionability timeout.
How do I wait for a button to become clickable? ⌄
Playwright auto-waits for actionability (visible, enabled, stable) on every action — no explicit wait needed. Selenium needs: new WebDriverWait(driver, Duration.ofSeconds(10)).until(ExpectedConditions.elementToBeClickable(locator));.
